## Architecture Decisions

1. **Direct SDKs over LangChain** ‚Äî Used `anthropic` + `openai` SDKs directly to avoid 30-50MB Lambda deployment bloat and 2-4s cold start penalty
2. **Lazy imports** ‚Äî AI SDK imports happen inside method calls so Lambda cold starts don't pay the cost if AI isn't used
3. **Config in existing table** ‚Äî Model configs stored as `pk=CONFIG, sk=ai_models` in existing `config` DynamoDB table, no new tables needed
4. **Backward compatible** ‚Äî `chat(message, session_id, user_id)` still works; `conversation_history` and `model_id` default to `None`
