## Architecture Decisions

1. **Reused existing DynamoDB config structure** — Custom models stored alongside registry overrides in `pk=CONFIG, sk=ai_models` → no new tables
2. **No animated backgrounds** — Unlike ToStructure's standalone page with gradient animations, this is embedded in the Settings page tabs, so kept styling consistent with other tabs (MUI default dark theme)
3. **JSON config textarea** — Same pattern as ToStructure: single JSON blob for api_key, temperature, max_tokens instead of individual form fields. Simpler and more flexible.
4. **Provider gradient icons** — Visual differentiation per provider matching ToStructure's `getProviderConfig()` pattern
